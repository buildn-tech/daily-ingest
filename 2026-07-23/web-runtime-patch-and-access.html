security release2026-07-20

Next.js July 2026 Security Release

What it is
Next.js published fixes in 16.2.11 and 15.5.21 for Image Optimization CPU exhaustion, Edge Server Action memory use, and response-body cache confusion.
Why it matters
The release turns a broad framework risk into concrete upgrade targets, including issues relevant to App Router, Server Actions, and server-side fetch.
Buildn / client use
Inventory client Next.js versions, identify exposed image optimization and Edge paths, update a staging property, and execute route-level smoke tests before production rollout.
Caveat
Impact depends on configuration and route usage; do not claim exposure without checking each deployed application.

Follow-up action: Open a patch ticket today with owner, deployed version, target version, test route, and rollout window.

notice2026-07-22

Upcoming GHES change impacting uploading support bundles

What it is
From August 18, GitHub will reject command-line support-bundle uploads from GHES appliances below specified patched releases.
Why it matters
Supportability is an operational dependency: an appliance that cannot upload diagnostic bundles is harder to recover precisely when an incident needs vendor help.
Buildn / client use
For any self-hosted GitHub client, record the appliance version, required patch line, owner, maintenance window, and a post-upgrade support-bundle test.
Caveat
This applies to GitHub Enterprise Server appliances, not GitHub.com organizations.

Follow-up action: Check GHES inventory and schedule required patching before the August 18 enforcement date.

release2026-07-09

R2 Data Catalog now supports read-only API tokens

What it is
Cloudflare R2 Data Catalog now accepts tokens scoped to catalog and storage read operations, while writes retain read-write requirements.
Why it matters
Analytics and retrieval clients no longer need a write-capable credential merely to list tables or query data.
Buildn / client use
Split ingestion and query identities for client reporting or RAG pipelines; issue read-only credentials to DuckDB, PyIceberg, or R2 SQL consumers.
Caveat
Vended credentials inherit R2 storage permissions, so catalog scope alone does not make a token read-only.

Follow-up action: Audit catalog consumers and replace every query-only credential with storage-and-catalog read-only scope.

release2026-07-20

View total SQLite storage for Durable Object namespaces

What it is
Cloudflare added an hourly Total storage chart for SQLite-backed Durable Object namespaces.
Why it matters
Stateful agent and collaboration workflows can fail quietly through retention and growth; an aggregate time series makes cleanup assumptions testable.
Buildn / client use
Add storage growth to the operational review for Durable Object–backed sessions, queues, or client workspaces; define retention and an alert threshold.
Caveat
The chart is aggregate-only and unavailable for legacy key-value storage namespaces.

Follow-up action: Review the largest stateful namespace, document retention behavior, and set a weekly growth check.
