Agent Sandbox + Tool Controls

The useful pattern is not “give an agent more tools.” Put tool exposure, credentials, and execution harnesses behind independent, replaceable boundaries. That makes a client pilot easier to audit and easier to stop.

Signal map

Selection: This is a fresh control-surface lane, selected after several days centered on evaluation and adoption: new releases make credential injection, harness substitution, and MCP exposure explicit configuration choices rather than hidden framework behavior.

Dossier cards

release2026-07-22

Agents SDK reduces MCP schema conversion and adds exposure controls

What it is
Cloudflare added a switch to opt out of Think’s automatic MCP tool exposure and direct host APIs for its Code Mode runtime.
Why it matters
Automatic tool discovery is convenient but expands an agent’s effective authority. An explicit opt-in creates a review point.
Buildn / client use
Use a per-workflow allowlist: start with read-only tools, add an action tool only with an owner and test case, then log every call.
Caveat
A framework flag is not a security boundary by itself; enforce authorization and data scope at each tool endpoint.

Follow-up action: Prototype one MCP workflow with `includeMcpTools = false`; add only the two tools required for its acceptance test.

releaseunknown

Safely inject credentials in HTTP headers with Vercel Sandbox

What it is
Vercel Sandbox can inject outbound HTTP headers at the network-policy layer, so sandboxed code calls an approved service without receiving the raw secret.
Why it matters
It separates agent runtime access from credential possession and supports a setup phase followed by deny-all execution.
Buildn / client use
For repository analysis or agent testing, allow credentials only for a short clone/bootstrap phase; remove egress and secrets before untrusted commands run.
Caveat
Header injection still grants capability to the sandbox for allowed destinations; constrain hostnames, paths where possible, and timeouts.

Follow-up action: Turn an existing sandbox policy into a two-phase policy: authenticated setup, then deny-all execution.

releaseunknown

Program Claude Code, Codex, Pi and other agent harnesses with AI SDK

What it is
AI SDK 7’s experimental HarnessAgent presents one session API around Claude Code, Codex, and Pi adapters.
Why it matters
Treating the harness as a replaceable implementation lets a workflow retain its policy, tools, skills, and acceptance checks while evaluation compares runners.
Buildn / client use
Build a small harness bake-off around a real bounded task: same repo snapshot, tools, budget, sandbox, and test command; compare completed work, cost, and cleanup.
Caveat
The adapters are experimental and may break; avoid making the abstraction itself a production dependency without pinning and a fallback.

Follow-up action: Pick one repetitive repair task and run it through two harnesses under identical limits.

release notes2026-07-22

Gemini Enterprise supports unauthenticated Custom MCP Server data stores (Preview)

What it is
Google Cloud’s release notes announce preview support for unauthenticated Custom MCP Server data stores in Gemini Enterprise.
Why it matters
This is a sharp reminder that “easy connector setup” can turn an unauthenticated endpoint into part of a business answer path.
Buildn / client use
Before connecting any client knowledge endpoint, classify it public/internal/restricted and require an authenticated retrieval path for internal material.
Caveat
The search result describes a preview feature only; confirm exact product behavior and tenancy controls in the current documentation before use.

Follow-up action: Add “connector authentication and audience” to the agent integration review checklist.
