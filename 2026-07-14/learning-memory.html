Daily Ingest Dossier — Learning + Memory

Today’s memory work converges on a useful operating point: memory needs structure, provenance and an active policy—not just better retrieval. The most practical next step is a safe reminder-only layer that cites durable project state and stays silent unless it has something material to add.

AutoMem: Automated Learning of Memory as a Cognitive Skill

papermemory policy1 Jul
What it isA framework that optimizes memory interface structure and an agent’s ability to manage files as memory actions.
Why it mattersIt separates “memory skill” from the underlying task agent, rather than treating memory as fixed plumbing.
Buildn use caseEvaluate Buildn memory schemas, write rules and retrieval behavior as an independent agent subsystem.
Useful nowThe paper reports 2–4× gains from memory-only optimization in long-horizon game environments.

Follow-up

Save — Extract a review rubric: what to store, how to name it, when to retrieve it and how to retire it.

Watch-out: Results are preprint claims in games; do not transfer the magnitude directly to production work.

NapMem: Active Memory Navigation

paperprovenance7 Jul
What it isA provenance-linked pyramid from raw conversations through typed records, topic tracks and user profiles, exposed as tools.
Why it mattersIt makes source drill-down and choice of granularity explicit agent actions.
Buildn use caseDesign project memory so summaries link back to decisions, evidence and original notes.
Useful nowIt offers a concrete alternative to dumping every fact into flat vector retrieval.

Follow-up

Open — Map one client/project archive into four levels: raw, record, track, profile.

Watch-out: The reinforcement-learning method may be heavier than the information architecture alone warrants.

Proactive Memory Agent for Long-Horizon Agents

paperselective reminders9 Jul
What it isA separate memory agent watches an action trajectory and injects a grounded reminder only when needed.
Why it mattersThe paper frames long-run failure as behavioral state decay, and shows selective reminders outperform constant injection in its tests.
Buildn use caseAdd a low-frequency “state check” agent to lengthy workflows: restate goals, constraints, prior failures and next gate.
Useful nowReported pass@1 gains on Terminal-Bench and τ²-Bench make intervention timing worth testing.

Follow-up

Try — Prototype a reminder-only reviewer that cannot act, write or call external tools.

Watch-out: Bad reminders can distract or anchor the action agent; require citations to saved project state.

CO-LMLM: Continuous-Query Limited Memory Language Models

paperexternal knowledge8 Jul
What it isA limited-memory language model design using vector queries over human-readable, attributable external factual memory.
Why it mattersIt argues for knowledge control and attribution instead of burying facts solely in weights.
Buildn use caseUse the design as a north star: retrieved project knowledge should be inspectable, attributable and updateable.
Useful nowIt is useful architecture vocabulary for deciding where Buildn facts live and how agents cite them.

Follow-up

Save — Add source URL and freshness fields to any durable memory record created by an agent.

Watch-out: This is a pretraining architecture, not a drop-in replacement for ordinary RAG.

TF-Engram: SSD-Backed Memory for LLMs

papermemory systems8 Jul
What it isA train-free phrase-memory system spanning GPU, DRAM and SSD, with predictive prefetching for inference.
Why it mattersIt treats durable external memory as a systems component with tiering and latency trade-offs.
Buildn use caseApply the analogy to storage policy: hot context, structured working memory and a cheap source archive.
Useful nowA useful counterweight to “put everything in the context window” designs.

Follow-up

Monitor — Do not build this; capture the tiered-memory principle in next architecture discussion.

Watch-out: Reported performance is on a small model and the implementation details may not generalize.
