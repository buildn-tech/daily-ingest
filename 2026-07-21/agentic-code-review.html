repo2026-07-17

Sashiko — agentic review of Linux kernel code changes

What
An Apache-2.0 Rust system that ingests patches from mailing lists, GitHub, GitLab, or local Git and produces structured review findings.
Why it matters
Its reported 53.6% bug detection on a historical kernel sample is less important than the explicit flow: intent, implementation, control flow, resources, locking, security, dedupe, verification.
Buildn / client use
Turn the stages into a PR template: run cheap structural checks first, then invoke a premium reviewer only for surviving concerns.
Caveat
The project reports probabilistic outputs and roughly 20% false positives in limited manual review; never auto-merge.

Follow-up: Borrow its staged review design for high-risk Buildn/client pull requests.

Buildn / client use
