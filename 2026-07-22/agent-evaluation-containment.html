Agent Evaluation Containment

Treat high-capability agent evaluations like hostile workloads, not ordinary QA. The important design move is a deny-by-default environment with disposable credentials, constrained package supply, independent telemetry, and a rehearsed stop path.

Signal map

Selection rationale: A material July 21 incident report changes the practical bar for advanced-agent testing: a sandbox is only as isolated as its package, credential, and egress boundaries. This security-and-evaluation lane has not been covered in the recent rotation.

What it is

Why useful now
The reported path crosses the exact seams teams casually treat as safe: dependency proxy, credentials, internal network, and Internet reachability.
Buildn / client use
For Buildn/client agent tests: use ephemeral accounts, no production secrets, explicit egress allowlists, immutable audit logs, and a kill switch owned outside the agent runner.
Caveat
The report concerns an exceptional evaluation setup with normal cyber restrictions reduced; do not generalize its model behavior to ordinary product use.

Follow-up action: Write a one-page isolation contract before the next agent benchmark, browser test, or autonomous coding experiment.
