Daily Ingest Dossier — Delivery Telemetry: Signal, Cost, and Latency

The useful shift is not more traces; it is telemetry that separates a bad cohort from a bad average and a costly span from a useful one. Treat observability configuration as a delivery budget: instrument the boundary, inspect the distribution, then remove noise.

Dossier cards

release2026-07-16

Heatmaps for Application Metrics

What: Distribution heatmaps for Sentry Application Metrics, including Custom Dashboards.

Why it matters: Averages hide tail latency and cohort divergence; heatmaps expose both over time.

Buildn / client use

Add one heatmap for request duration and one for queue wait on a client critical path.

Follow-up

Open the metrics dataset and choose a distribution metric before adding any new dashboard panel.

Caveat: Useful only when the metric is emitted as a distribution; a count alone cannot show latency shape.

release2026-07-15

Dart/Flutter SDK: gRPC Integration

What: sentry_grpc adds client-call tracing, breadcrumbs, trace propagation, and failed-call capture for Dart/Flutter.

Why it matters: Mobile-to-backend hops stop being an uncorrelated blind spot.

Buildn / client use

Trace a Flutter client request through the gRPC backend when diagnosing client-specific failure reports.

Follow-up

Test SentryGrpcInterceptor on a staging client and validate trace continuity at the backend.

Caveat: Only calls routed through the configured gRPC client interceptor are automatically instrumented.

release2026-07-08

Low-Value Spans (Beta)

What: An issue category that identifies high-volume, low-signal spans with projected 30-day volume, cost, duration, and removal guidance.

Why it matters: Instrumentation has a carrying cost; noisy automatic spans can obscure incidents and inflate storage.

Buildn / client use

Run a monthly tracing hygiene pass before expanding agent or workflow tracing.

Follow-up

Filter issue.category:configuration and remove one low-value auto-instrumented span in a noncritical flow.

Caveat: Open beta; Business and Enterprise only, and these issues intentionally do not notify by default.

release2026-07-07

Dashboard Equations for Application Metrics

What: Custom dashboards can calculate equations across Application Metrics.

Why it matters: Operational questions are often ratios—error share, cache efficiency, or cost per completed job—not raw series.

Buildn / client use

Define a single delivery health ratio per client service instead of a dense generic dashboard.

Follow-up

Prototype error-rate and successful-job-per-worker-minute formulas against an existing dashboard.

Caveat: Equations improve clarity only when the numerator and denominator share a meaningful time window.
