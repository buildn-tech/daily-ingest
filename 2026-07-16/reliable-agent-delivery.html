releaseupdated 2026-07-15

Vercel CLI 55: explicit project scope

What changed
CLI 55 resolves a team before finding projects; non-interactive commands fail with missing_scope rather than guessing.
Buildn / client use
Set VERCEL_ORG_ID and VERCEL_PROJECT_ID in Buildn/client CI so deploy agents cannot link or deploy into the wrong team.

Caveat: Major version with breaking changes; scripts that relied on --yes or global team selection must be updated.

Next move: Pin v55 in a non-production pipeline and add explicit team/project variables.

releaseupdated 2026-07-15

Vercel Workflows trace-viewer minimap

What changed
A minimap exposes the whole workflow run, with pan, zoom, jump, and local access through the beta Workflow SDK web UI.
Buildn / client use
Use trace inspection as a release gate for multi-step agents: identify retries, long spans, and unexpected branches before expanding autonomy.

Caveat: This improves inspection, not correctness; retain structured logs, idempotency, and alerting.

Next move: Open one representative workflow trace and document its expected branches and maximum duration.

releaseupdated 2026-07-15

n8n 2.30.5 stable

What changed
The latest stable patch includes AI assistant/core fixes and ships SBOM, third-party-license, and OpenVEX artifacts.
Buildn / client use
For self-hosted client automations, treat upgrade review as a supply-chain checkpoint: validate the release artifacts and smoke-test credentials/webhooks.

Caveat: The beta line contains additional changes; do not promote it simply because it is newer.

Next move: Stage 2.30.5, test a credentialed workflow and webhook failure path, then schedule the production upgrade.
