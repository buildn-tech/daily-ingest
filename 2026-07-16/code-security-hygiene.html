GitHub Copilot app: /security-review

What changed
Runs a high-confidence review of in-flight local changes for injection, XSS, path traversal, insecure data handling, and weak crypto.
Buildn / client use
Add it as a pre-commit habit for Buildn/client changes; save verified findings as review evidence, not automatic fixes.

Caveat: Preview output needs human judgment; it complements code scanning, dependency review, and secrets controls.

Next move: Run it on one active client change and compare its findings with the PR review.

releaseupdated 2026-07-14

AI security detections on pull requests

What changed
AI-powered detections now appear in PRs for coverage gaps beyond CodeQL-supported languages and frameworks.
Buildn / client use
Pilot it on repositories with CodeQL default setup; triage its informational, AI-labelled findings in the existing PR ritual.

Caveat: Public preview; needs enterprise approval, GitHub Code Security, CodeQL default setup, Copilot licensing, and consumes AI credits.

Next move: Inventory eligible repositories and enable a two-week pilot on the most exposed one.

releaseupdated 2026-07-15

Secret scanning: categories and new detectors

What changed
GitHub added Resend/APIclub detection, default blocking for VolcEngine Ark keys, and a secret_category field in alert webhooks.
Buildn / client use
Route generic/AI-detected secrets differently from provider/custom patterns; use the category in a small incident workflow.

Caveat: Detection is only one layer: rotation, ownership, and prevention of sensitive config in repos still matter.

Next move: Confirm secret-scanning webhooks record category, repo, owner, and rotation status.
