Daily Ingest Dossier — Learning + Memory

Memory work is converging across two worlds: human learning research keeps reinforcing retrieval-plus-sleep dynamics, while agent memory papers now focus on consolidation, forgetting, contamination, and graph memory. The Buildn opportunity is to treat memory as a governed lifecycle, not an append-only vector store.

1. SCM: Sleep-Consolidated Memory for LLMs

  • source: https://arxiv.org/html/2604.20943v1
  • follow-up: save
What it isA research-preview architecture with limited working memory, value tagging, NREM/REM-style consolidation, intentional forgetting, and a self-model.

2. FSFM selective forgetting framework

turn into taskpaper2026-04-23
  • source: https://arxiv.org/html/2604.20300v2
  • follow-up: turn into task
What it isA biologically inspired framework for passive decay, active deletion, safety-triggered forgetting, and adaptive memory pruning in LLM agents.
Why it mattersForgetting is positioned as efficiency, quality, security, and privacy infrastructure.
Use cases for Abdelrahman / BuildnTurn into a Buildn memory-policy spec: retain, decay, quarantine, delete, and explain memory decisions.
Why useful nowMemory-enabled agents increase liability unless deletion and staleness are first-class.

Caveat

Enterprise case-study claims may not transfer directly to smaller systems.

3. Remembering More, Risking More

openpaper2026-05-18
  • source: https://arxiv.org/html/2605.17830v1
  • follow-up: open
What it isA study of temporal memory contamination where benign accumulated memory later causes unsafe behavior in unrelated tasks.
Why it mattersIt shows memory risk can grow without an attacker; safety must be evaluated longitudinally.
Use cases for Abdelrahman / BuildnAdd retrieval-state monitoring and memory isolation tests to any Buildn long-term agent memory rollout.
Why useful nowAs agents persist across clients/projects, cross-session contamination becomes a practical risk.

Caveat

Evaluation protocols are emerging; use as a warning pattern, not final standard.

5. Cognee persistent memory layer guide

tryblog2026-05-20
  • source: https://www.cognee.ai/blog/guides/building-an-ai-agent-best-persistent-memory-layer
  • follow-up: try
What it isA 2026 guide arguing for graph-vector hybrid memory layers for production agents that need relationships, provenance, and multi-hop reasoning.

Use cases for Abdelrahman / BuildnPrototype graph-backed memory for client/project facts: people, decisions, artifacts, dates, and dependencies.
Why useful nowBuildn’s artifact library and agents would benefit from relationship-aware recall.

Caveat

Graph memory adds schema and maintenance overhead.
